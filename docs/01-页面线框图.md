# 页面线框图

本文档不追求视觉稿，而是明确每个页面的模块、优先级、操作入口和关键状态，便于后续出 Figma 或直接实现前端页面。

## 1. 全局导航

### 一级导航

1. 工作台
2. 资料投放
3. 资产中心
4. 内容日历
5. 生成记录
6. 设置

### 全局顶部栏

1. 当前品牌空间切换器
2. 全局搜索
3. 快速上传按钮
4. 待确认资产提醒
5. 用户菜单

### 全局状态规则

1. 待确认资产数量需要在导航上持续可见。
2. 日历相关入口优先级最高，桌面端建议固定在主导航前部。
3. 所有生成类结果都要能追溯到来源资产和生成时间。

## 2. 工作台

### 页面目标

让用户一进系统就知道今天该做什么，以及系统已经准备好了什么。

### 结构草图

```text
+----------------------------------------------------------------------------------+
| 顶部：品牌空间 | 日期范围 | 快速上传 | 生成今日内容包                              |
+----------------------------------------------------------------------------------+
| 左侧主区                                                                      |
| [今日推荐内容]                                                                 |
| - 今日主题                                                                     |
| - 推荐平台                                                                     |
| - 内容目标                                                                     |
| - 已生成内容包状态                                                             |
| - 缺失素材提醒                                                                 |
|                                                                                |
| [本周任务]                                                                     |
| - 待确认资产                                                                   |
| - 待补活动资料                                                                 |
| - 待审核内容                                                                   |
|                                                                                |
| [最近上传资料]                                                                 |
| - 文件名 / 类型 / 识别结果 / 是否已入库                                       |
|                                                                                |
| 右侧侧栏                                                                       |
| [活动提醒]                                                                     |
| [本周日历缩略图]                                                               |
| [最新生成物料]                                                                 |
+----------------------------------------------------------------------------------+
```

### 关键动作

1. 一键跳转到“今天”的内容日历详情。
2. 快速继续上一次未完成的资料确认。
3. 直接从工作台上传新资料。

## 3. 品牌初始化页

### 页面目标

在 10 分钟内完成品牌空间初始化，并输出一份可作为系统长期约束的品牌底稿。

### 页面分步

1. 基础信息
2. 品牌表达
3. 产品与平台
4. 视觉资产
5. 确认生成

### 结构草图

```text
+----------------------------------------------------------------------------------+
| 左侧步骤导航                                                                    |
| 1 基础信息                                                                      |
| 2 品牌语气                                                                      |
| 3 产品概况                                                                      |
| 4 平台偏好                                                                      |
| 5 视觉素材                                                                      |
+------------------------------------+---------------------------------------------+
| 表单区                             | 右侧实时输出区                              |
| 品牌名                             | 品牌画像草稿                                |
| 定位                               | 核心主张                                    |
| 目标用户                           | 品牌语气标签                                |
| 不希望被写成什么                   | 禁用方向                                    |
| 常用平台                           | 初始内容支柱                                |
| 上传 logo / 视觉参考               | 初始周历建议                                |
+------------------------------------+---------------------------------------------+
| 底部：保存草稿 | 生成品牌底稿 | 完成初始化                                      |
+----------------------------------------------------------------------------------+
```

### 关键规则

1. 用户确认前，不写入不可逆的品牌核心定义。
2. 页面右侧的品牌画像需要可解释，明确哪些内容来自用户输入，哪些内容来自系统建议。
3. 初始化完成后产出两份对象：`brand_profile` 和 `brand_ruleset`。

## 4. 资料投放页

### 页面目标

把复杂录入过程降到最低，像“投递箱”而不是后台表单。

### 结构草图

```text
+----------------------------------------------------------------------------------+
| 顶部：拖拽上传区 / 粘贴文本 / 腾讯文档链接 / 语音转文字 / 图片视频上传            |
+----------------------------------------------------------------------------------+
| 左侧：新投递列表                       | 右侧：系统识别反馈                          |
| - 文件1                                | 资料类型：产品 / 活动 / 素材                |
| - 文件2                                | 置信度：85%                                 |
| - 文本片段                             | 建议归类：某产品线                          |
|                                        | 建议标签：上新 / 用户案例 / 小红书          |
|                                        | 是否进入核心资产：建议否                    |
|                                        | 是否需要确认：是                            |
+----------------------------------------+--------------------------------------------+
| 下方：批量动作                                                                   |
| 重新识别 | 批量确认 | 进入待确认区 | 直接入素材库                                |
+----------------------------------------------------------------------------------+
```

### 页面状态

1. 上传中
2. 解析中
3. 已识别待确认
4. 已自动入库
5. 识别失败待人工处理

### 关键交互

1. 对低置信度内容默认进入待确认资产。
2. 对纯素材类内容支持批量自动入库。
3. 每条资料都要保留原始文本或原始文件引用。

## 5. 资产中心

### 页面目标

把“知识库”做成可操作资产库，不只是浏览页。

### 结构草图

```text
+----------------------------------------------------------------------------------+
| 顶部：搜索 | 标签筛选 | 来源筛选 | 时间筛选 | 新建资产                            |
+----------------------------------------------------------------------------------+
| Tabs: 品牌资产 | 产品资产 | 活动资产 | 素材资产 | 待确认资产                         |
+----------------------------------------------------------------------------------+
| 左侧列表                                                                          |
| - 资产名称                                                                        |
| - 类型                                                                            |
| - 最近更新时间                                                                    |
| - 关联数量                                                                        |
|                                                                                    |
| 右侧详情                                                                          |
| - 核心字段                                                                        |
| - 来源资料                                                                        |
| - 关联产品/活动/内容                                                              |
| - 可用于哪些平台                                                                  |
| - 是否为核心定义                                                                  |
| - 编辑历史                                                                        |
+----------------------------------------------------------------------------------+
```

### 每个 Tab 的重点

1. 品牌资产：可编辑，但核心字段必须走确认流程。
2. 产品资产：强调卖点、场景、FAQ、证据链。
3. 活动资产：强调时间、目标、关联产品、素材缺口。
4. 素材资产：强调格式、标签、平台适配、来源。
5. 待确认资产：强调识别原因、建议归类、确认动作。

## 8. 设置页

### 核心模块

1. 品牌空间信息
2. 成员与权限
3. 平台偏好
4. 外部连接
5. 导出格式
6. 自动生成频率

### 关键限制

1. 品牌核心规则仅管理员可编辑。
2. 外部连接失败不能影响主链路上传和生成。
3. 自动生成频率要支持关闭，避免用户失控感。
