# 产品规格说明书

## 1. 目标

BrandAgent 的 MVP 目标是把“上传资料 -> 沉淀资产 -> 日历排期 -> 生成内容包 -> 下载交付”打通成一个最短工作闭环。

### 成功标准

1. 新用户在 10 分钟内完成品牌初始化。
2. 用户可以在 3 分钟内上传一批资料并看到分类结果。
3. 用户可以在日历中打开任意一天，查看已生成内容包并下载。

## 2. 范围

### In Scope

1. 品牌空间初始化
2. 资料上传与识别反馈
3. 资产中心展示
4. 内容日历浏览与单日详情
5. 内容包查看与导出入口
6. 基础角色和平台配置展示

### Out of Scope

1. 实际接入腾讯文档 API
2. 真正的 AI 模型推理和向量检索
3. 自动发布到第三方平台
4. 真实团队协作审批流

## 3. 关键用户故事

### US-01 品牌初始化

作为品牌负责人，我希望在首次进入时完成品牌初始化，这样系统才能形成可持续使用的品牌规则。

验收标准：

1. 可以录入品牌名、定位、目标用户、语气、禁用方向、平台偏好。
2. 可以看到系统生成的品牌画像和规则卡预览。
3. 核心品牌规则必须需要用户确认后才生效。

### US-02 资料投放

作为内容运营，我希望像投递箱一样把文本、链接、图片和活动资料扔进系统，让系统先帮我识别和建议归类。

验收标准：

1. 页面提供上传、粘贴文本和输入链接入口。
2. 每条资料都显示类型建议、置信度和入库建议。
3. 低置信度内容会进入待确认区。

### US-03 资产沉淀

作为市场负责人，我希望在一个地方查看品牌、产品、活动和素材资产，并知道这些资产与内容生产的关系。

验收标准：

1. 资产中心至少包含品牌、产品、活动、素材、待确认五个标签页。
2. 资产详情展示来源、关联数量、状态和关键字段。
3. 核心品牌资产和普通素材资产在状态上有明显区分。

### US-04 日历排期

作为创始人，我希望进入系统后先看见今天该发什么，而不是先维护一堆知识库。

验收标准：

1. 日历页支持展示当天主题、平台、目标、状态。
2. 对缺素材的计划有明确提醒。
3. 用户可以从日历进入单日内容包详情。

### US-05 内容包下载

作为代运营服务商，我希望打开某一天后直接看到标题、正文、海报文案、视频脚本和导出入口。

验收标准：

1. 物料详情页展示标题候选、正文、海报文案、镜头建议、CTA。
2. 页面显示关联资产和当前版本信息。
3. 页面保留 md、docx、pdf、txt 导出入口位置。

## 4. 功能规格

### F-01 营销首页

1. 展示产品定位、核心价值、关键页面入口。
2. 提供进入体验空间的 CTA。

### F-02 工作台

1. 展示今日推荐内容、本周任务、最近上传、最新生成物料、活动提醒。
2. 支持跳转到日历、上传和初始化流程。

### F-03 品牌初始化

1. 使用分步式表单。
2. 右侧实时展示品牌画像草稿和规则卡输出。
3. 生成品牌底稿前不得直接覆盖已确认规则。

### F-04 资料投放

1. 支持文件、文本、链接三种主要入口。
2. 支持展示系统识别反馈和批量动作区。
3. 提示哪些资料会直接进入素材库，哪些需要确认。

### F-05 资产中心

1. 提供 tab 分类浏览。
2. 提供搜索、筛选和关系展示区。
3. 支持展示待确认资产的处理入口。

### F-06 内容日历

1. 支持周/月心智的主界面布局。
2. 每个日期块展示主题、平台、状态。
3. 支持打开单日详情，查看 brief、缺口素材、生成结果。

### F-07 内容包详情

1. 展示当前版本内容和导出入口。
2. 展示关联资产与生成记录。
3. 保留重新生成和平台改写动作入口。

### F-08 设置

1. 展示品牌空间、成员角色、平台偏好和自动生成频率。
2. 强调品牌核心规则仅管理员可编辑。

## 5. 非功能要求

1. 核心业务对象都需带 `workspace_id` 概念。
2. 生成结果需要保留资产快照，满足可追溯要求。
3. UI 需要同时支持桌面端和移动端浏览。
4. 项目可以在 Vercel 预览环境直接构建部署。

## 6. 前端实现边界

当前骨架使用 mock data 驱动，以保证前端页面结构、信息层级和测试基线先成立。真实后端接入时建议按以下模块替换：

1. `src/lib/domain.ts` 中的规则函数替换为服务层逻辑。
2. `src/lib/mock-data.ts` 替换为 API 返回。
3. 页面中的状态模块替换为 server actions 或 BFF。
