## 当前默认假设

1. 产品第一期为多租户 SaaS，每个品牌空间对应一个 workspace。
2. 首期重点是“上传整理 + 日历规划 + 单日内容包生成 + 本地下载”。
3. 外部平台同步、自动发布、效果回流先不进入 P0 的强依赖链路。
4. AI 生成结果默认需要可追溯，保留来源资料和生成任务记录。

## 建议下一步

1. 先按 `01-页面线框图` 出低保真原型。
2. 按 `03-数据库表设计` 搭最小表结构和对象接口。
3. 以 `02-后台信息架构` 为基础拆成 ingestion、asset、calendar、generation 四个服务模块。
