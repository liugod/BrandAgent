# 数据库表设计

本文档按 MVP 优先，给出建议的核心表结构。目标不是一次性设计到完美，而是先保证主链路能稳跑。

## 1. 建模原则

1. 多租户隔离以 `workspace_id` 为主键外键链路核心。
2. 品牌核心定义与系统建议要分层存储，避免被自动覆盖。
3. 原始资料、结构化资产、日历计划、生成结果四层分开。
4. 生成结果采用版本化，而不是覆盖写。

## 8. 关键约束总结

1. 所有业务表都必须带 `workspace_id`。
2. 品牌核心规则通过 `brand_rules` 版本化，不允许系统直接覆盖当前启用版本。
3. `source_items` 永远保留原始内容引用，不做物理覆盖。
4. `content_package_versions` 必须保存引用资产快照，保证可追溯。
5. 自动分类低置信度结果必须通过 `asset_confirmations` 留痕。
